NOTIFICATION SYSTEM - QUICK START GUIDE
========================================

## What's New

Your CITAS system now automatically sends notifications for:

1. 📤 **New Thesis Available** - When approved thesis is uploaded
2. 🗑️ **Thesis Deleted** - When thesis is removed
3. 📋 **Access Request** - When student requests access (admin notified)
4. ✅ **Access Approved** - When access is granted (student notified)

## Quick Start (3 Steps)

### Step 1: Initialize Database
```
1. Open browser → http://localhost/ctrws-fix/init_notifications.php
2. Wait for message: "=== NOTIFICATION SYSTEM READY ==="
3. Done! Database is set up.
```

### Step 2: Test Locally
```
1. Upload a thesis with status=Approved → Students get notification
2. Delete a thesis → All users get notification
3. Request access as student → Admin gets notification
4. Approve request as admin → Student gets notification
```

### Step 3: Deploy to Hostinger
```
1. Upload 5 modified files via File Manager:
   - admin_includes/admin_add_thesis.php
   - admin_includes/admin_delete_thesis.php
   - admin_includes/admin_approve_access.php
   - client_includes/request_access_code.php
   - client_includes/create_notification.php

2. Upload initialization script:
   - init_notifications.php

3. Run: https://yourdomain.com/ctrws-fix/init_notifications.php

4. Test on your live site
```

## Notification Types at a Glance

### 📤 thesis_upload
- **When:** Admin uploads thesis with "Approved" status
- **Who:** All students & instructors
- **Says:** "A new thesis '[Title]' by [Author] is now available for viewing."

### 🗑️ thesis_deleted
- **When:** Admin deletes any thesis
- **Who:** All active users
- **Says:** "The thesis '[Title]' has been removed from the system."

### 📋 access_request
- **When:** Student/instructor requests access
- **Who:** Admin only
- **Says:** "[User Name] has requested access to '[Title]'"

### ✅ access_approved
- **When:** Admin approves request
- **Who:** User who requested
- **Says:** "Your request to access '[Title]' has been approved."

## Troubleshooting

**Notifications not showing?**
1. Check: http://localhost/ctrws-fix/init_notifications.php
2. Clear browser cache (Ctrl+Shift+Delete)
3. Reload page and check bell icon
4. Check PHP error logs

**Timestamp looks wrong?**
- Verify server time matches database timezone
- Check php.ini for correct timezone setting

**Missing a specific notification?**
1. Check if prerequisite action was done
2. Look in: Settings → Logs → PHP errors
3. Query database: SELECT * FROM notifications
4. Verify admin user exists and is active

## Database Check

To verify notifications are working, run in your database client:

```sql
SELECT * FROM notifications ORDER BY created_at DESC LIMIT 10;
```

You should see recent notifications with:
- user_id (who received it)
- type (thesis_upload, thesis_deleted, etc.)
- title & message (notification content)
- is_read (0 for unread, 1 for read)
- created_at (when it was created)

## What Happens Under the Hood

1. **Event Triggers**
   - When thesis uploaded/deleted/requested/approved
   - PHP code creates notification entry in database

2. **Frontend Polling**
   - Browser checks every 30 seconds for new notifications
   - Uses JavaScript + AJAX (no page refresh needed)

3. **Real-Time Display**
   - Bell icon updates with unread count
   - Dropdown shows latest notifications
   - Time displays as "5 minutes ago", "just now", etc.

4. **User Action**
   - Click notification → marks as read
   - Notification stays but no longer highlighted
