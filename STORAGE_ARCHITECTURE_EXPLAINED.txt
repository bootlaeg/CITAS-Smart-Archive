PROFILE PICTURE STORAGE - VISUAL OVERVIEW
==========================================

CURRENT SYSTEM ARCHITECTURE:
============================

User Interface (my_profile.php)
    ↓
    [Edit Profile Modal]
    [Select image file]
    ↓
Backend Handler (client_includes/update_profile.php)
    ↓
    ├─ Validates image (type, size)
    ├─ Creates uploads/profile_pictures folder if needed
    ├─ Saves file: uploads/profile_pictures/profile_123_1704067200.jpg
    └─ Deletes old image if exists
    ↓
Database Entry
    ├─ Table: users
    ├─ Column: profile_picture
    └─ Stored value: "uploads/profile_pictures/profile_123_1704067200.jpg"


WHEN USER VIEWS PROFILE:
========================

Browser requests: my_profile.php
    ↓
PHP retrieves user data from database
    ├─ Gets profile_picture path: "uploads/profile_pictures/profile_123_1704067200.jpg"
    ├─ Generates HTML: <img src="uploads/profile_pictures/profile_123_1704067200.jpg">
    └─ Browser displays image
    ↓
User sees: Beautiful circular profile picture ✓

KEY ADVANTAGES OF THIS APPROACH:
================================

✓ File System Storage (not database):
  - Faster loading (no database query for image)
  - Smaller database size
  - Better for backups
  - Images cache naturally in browser
  - Works on all hosting providers

✓ Path Stored in Database:
  - Easy to track which user has which picture
  - Can delete old pictures when updating
  - Flexible (can move to CDN later if needed)
  - Simple to implement


DATA FLOW WHEN UPLOADING:
=========================

1. User selects image in browser
   File: profile_123_original_name.png

2. JavaScript preview shows image instantly
   FileReader API reads locally (no upload yet)

3. User clicks "Save Changes"
   FormData with image sent to update_profile.php

4. Server validates:
   ✓ File type (JPG, PNG, GIF, WebP only)
   ✓ File size (max 5MB)
   ✓ File MIME type

5. Server saves file:
   Path: uploads/profile_pictures/profile_1_1704067200.png
   Filename: profile_{user_id}_{timestamp}.{ext}

6. Server stores path in database:
   UPDATE users SET profile_picture = 'uploads/profile_pictures/profile_1_1704067200.png'

7. Server deletes old image if exists
   File deleted: uploads/profile_pictures/profile_1_1704066000.png

8. Response sent to browser:
   {"success": true, "message": "Profile updated successfully!"}

9. Page reloads, shows new profile picture


SAFETY & SECURITY:
==================

✓ File type validation (MIME type check)
✓ File size limit (5MB max)
✓ Unique filenames (prevents overwrite)
✓ Old files deleted automatically
✓ File extension matched to MIME type
✓ Relative paths (no absolute path exposure)
